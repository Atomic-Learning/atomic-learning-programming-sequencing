There are some special considerations when sequencing programming pages in the Atomic Learning platform. These tend to derive from the fact that there are many different programming languages and that programming tools can be applied across many different disciplines.

Language-Agnostic Pages

Many topics in programming will be applicable across many different programming languages. To avoid repetition for learners, pages covering these broad concepts should be introduced in language-agnostic pages. These pages aim to provide an understanding of the fundamental principles of programming that transcend specific languages (such as what a variable is, or what a function is).

In many cases, further pages will be needed to provide language-specific details, and these would use the language-agnostic page as a prerequisite. However, these pages would not require reintroduction of the underlying concepts and behaviours and would focus solely on language-specific syntax and behaviours.

The diagram above shows how a theoretical series of related programming pages could be sequenced. This type of relationship has two key advantages:

  • The foundational language-agnostic concepts only need to be written once, and can be reused across many different language-specific pages.
  • If a learner already knows one programming language, learning a new one will only require learning the language-specific syntax and features of this new language.

Discipline-Specific Examples and Exercises

In most cases, the pages which teach the foundations of programming (either in a language-agnostic way or relating to a particular language) should use examples to demonstrate the concepts, features or syntax which are:

  • Simple: Learners should not experience unnecessary cognitive load understanding the situation being used as an example.
  • Minimal: Examples should be concise to minimise length.
  • Discipline-agnostic: Examples should be relevant to learners from various disciplines, avoiding domain-specific details that would otherwise require additional prerequisites.

However, it is encouraged to add additional pages which provide complex, extensive and discipline-specific content that use these core pages as prerequisites. This allows learners to see how the concept, feature or syntax they have just learned about in the abstract can be applied in real-world scenarios relevant to their discipline. This means examples which are used to teach the content in pages like Python: Functions are likely to be generic and minimal, such as adding two numbers together. This minimises the prerequisites of the page and the cognitive load required when learning these new concepts. When constructing a Learning Path, it is often good practice to include more complex or discipline-specific examples and exercises so learners can also see the applications of what they have learned in a more interesting and practical way.
